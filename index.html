Session

A session code is automatically created by Acty app native layer at start time.
Acty then communicates it to your web page by calling the Javascript function ACTYObjSetSession(value) that you MUST define in your web page (see Docs) .
The argument value to the function will hold such session code.
In this example, the function simply fills a form field.

Getting GPS

To exchange GPS with Acty native layer, you need to put in your web page a button (which requests the GPS info) and define a function to receive the GPS value.

The function MUST be named ACTYObjSetGPS (see Docs).
In this example page, the function simply fills a form field.

The button MUST call the ACTYObjGetGPS function (see Docs).
You don't find this function in this example page because it is injected at runtime in your web page by Acty app native layer.

Start Call

To initiate a call with your company from mobile, all that you need is a button that calls the ACTYObjCall function.

You don't find this function in this example page because it is injected at runtime in your web page by Acty app native layer.

DOCS

Function for calling Acty

ACTYObjCall() this function will put the user in the Acty queue

ACTYObjChat() this function will open the chat screen

ACTYObjGetGPS() this function will ask for the gps position (the ACTYObjSetGPS will be called in response)

Function called from Acty

ACTYObjSetSession(value) this function will be called when the session code is available (on each onload event)

ACTYObjSetGPS(pos) this function will be called when the gps pos is available (in ISO 6709 format)

ACTYObjSetUnreadMessages([{company_code: String, unread_messages: int}, ...]) this function will be called on each reload with the number of unread messages
